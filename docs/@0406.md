  Phase 3: 5-fold CV at +200 (statistical rigor)

  Goal: Train baseline/d3/d4/d5 with 5-fold CV to get tight CIs and test d5 > d4 significance.

  Pipeline changes needed:
  1. New script: scripts/kfold_split.py — split real images into 5 folds (stratified by species)
  2. Modified assembly: For each fold, copy fold's train split + add synthetic to train only
  3. Training: 4 configs × 5 folds = 20 training runs
  4. Evaluation: Each fold tested on its held-out set, aggregate predictions
  5. Analysis: Paired t-test on fold-level macro F1 (d4 vs d5, d5 vs baseline)

  Key design decisions:
  - The test set stays fixed (same as current) — folds are on the training data only (train+val reshuffled). This means each fold retrains on a different 80% of train+val, but
  tests on the same test set. Result: 5 F1 measurements per config on the same test set, capturing training variance.
  - Alternative: true 5-fold where test set rotates. This gives more test data but requires careful handling of augmentation leakage.

  Decision: Use rotating test set (true 5-fold). Each fold: train on 4 parts + synthetic, test on held-out 1 part.
  - Solves the core problem: ashtoni n=6 → test on all ~30 real images across folds
  - Paired t-test valid because within each fold d3/d4/d5 share same test set
  - Synthetic images ONLY added to training folds (no leakage)

  Deliverables: Paired t-test p-values, box plot of fold-level F1, final table for paper
