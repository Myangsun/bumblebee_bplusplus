Results 1: Classifier Performance Across Augmentation Methods
Setup: ResNet-50 trained on 4 dataset configurations, evaluated on 2,362 test images (16 species).
Key finding:
  - LLM-filtered synthetic (D5) achieves the best overall accuracy and best/tied-best rare species
  F1.
  - Filtering vs. unfiltered synthetic: +0.34% overall, with clearer gains on rare species
  (sandersoni 0.47→0.56, ashtoni 0.62→0.67).
  - Copy-paste (D3) remains competitive despite being a simpler method — it uses real morphology,
  which may explain why it performs well even without filtering.

Next steps:
  - Add B. flavidus: Third rare species (162 train images). B. flavidus F1 improved from 0.67→0.84 in D3 as an indirect effec,  augmenting ashtoni/sandersoni sharpened the model's decision boundaries,
  reducing flavidus misclassifications from 15→7 across multiple confusable species (citrinus, terricola, ashtoni). Since flavidus was not directly augmented, this suggests it could benefit further from targeted synthetic augmentation.
  - Statistical significance: Per-species F1 is unreliable with test sizes of 6 (ashtoni) and 10
  (sandersoni). Plan: (1) bootstrap confidence intervals on per-species metrics, (2) McNemar's test
  on the full 2,362-image paired predictions for overall accuracy, (3) consider multi-seed training
  (3 seeds) to report mean ± std.

Results 2: LLM Judge Pass/Fail Distribution Analysis
Setup: GPT-4o structured output evaluated 1,100 synthetic images (650 ashtoni, 450 sandersoni) using a two-stage blind identification + morphological scoring protocol.

Key finding — species-level generation quality differs dramatically:
  - B. ashtoni: 50% strict pass rate (325/650 passed all 3 criteria)
  - B. sandersoni: 93% strict pass rate (420/450 passed)

Failure analysis revealed correlations with viewpoint and background:
  - Viewpoint: B. ashtoni pass rate varies from 78% (lateral) to 20% (posterior). Lateral views
  expose the most diagnostic features (T4 banding, thorax coloration), while posterior views hide
  them.
  - Background: Yellow flower backgrounds correlate with lower pass rates (~29–40% for ashtoni vs.
  ~55–61% on green foliage). Hypothesis: yellow flowers may visually compete with the yellow T4
  tergite band that is the primary diagnostic feature — the judge may struggle to distinguish
  tergite coloration from surrounding yellow petals.

Next steps:
  - Remove backgrounds from failed synthetic images (replace with white/blank) and re-run the LLM judge. If pass rates increase on the same images, this confirms the background is confounding the judge's assessment (not just a generation quality issue). 
  - Regenerate with corrected scale: Some synthetic images show bumblebees that are
  disproportionately large or small relative to flowers.







Baseline: Flavidus 21/36 correct (58%), with 15 misclassifications spread across 7 species                                                    
  D3: Flavidus 29/36 correct (81%), only 7 misclassifications
                                                                                                                                                
  The key changes:                                                                                                                            
  - Flavidus→ashtoni confusion dropped (2→1)
  - Flavidus→terricola dropped (3→1)
  - Flavidus→citrinus dropped (4→2)
  - The ashtoni→flavidus false positive stayed at 1

  So it's not purely an ashtoni/sandersoni confusion effect — the improvement is broader. By augmenting ashtoni/sandersoni, the model learned
  better representations for those species, which tightened decision boundaries across multiple confusable species. Flavidus benefited because
  it was frequently confused with citrinus, terricola, and ashtoni in the baseline, and the improved model simply became better calibrated
  overall.

  Proposed Validation Pipeline

  Phase 1 — Expert binary filter (species verification)
  - Expert sees each synthetic image with full species metadata
  - Task: binary yes/no — "Is this a correct representation of [target species]?"
  - Fast (~5–10 sec/image), doesn't require MA-specific familiarity since metadata is provided

  Phase 2 — Automated morphological scoring
  - Expert-passed images go through the existing LLM judge validation pipeline
  - Scores 5 morphological features on 1–5 scale
  - Gives quantitative quality signal per image

  Phase 3 (addon) — Crowdsource re-validation
  - An additional layer on top of Phase 1+2, not a replacement
  - Crowdsource reviewers (e.g., iNaturalist community, undergrad assistants) independently validate the same images
  - Provides inter-annotator agreement statistics for the paper
  - Strengthens publishability: expert + automated + crowd consensus

  Options for Using Richard's 3 Hours

  Option A: Broad coverage — sample across all images
  - Review ~200–300 images stratified across both species, pass/fail, and viewpoints
  - Pro: Gives representative calibration of the LLM judge across the full distribution; computable Cohen's kappa
  - Con: Spread thin — may not deeply characterize any single failure mode

  Option B: Targeted — focus on LLM judge failures and borderline cases
  - LLM judge results already exist for all 1,100 images
  - Richard reviews ~200 B. ashtoni failures (the hard species, 50% fail rate) + ~50–100 passes as calibration
  - Pro: Maximizes information gain — expert attention goes where the automated filter is most uncertain; directly answers "is the LLM judge
  rejecting good images or keeping bad ones?"
  - Con: Skewed toward one species; less useful for validating sandersoni pipeline

  Option C: Volume-informed — run ablation first, then validate only what's needed
  - First train with 50/100/200/300 LLM-filtered images to find the accuracy elbow point
  - Then ask Richard to validate only the volume we actually need
  - Pro: Don't waste expert time on images that won't be used
  - Con: Delays expert involvement; ablation takes cluster time first

  My Suggestion

  Option B first, then Option C in parallel.

  1. Richard's 3 hours are most valuable on B. ashtoni failures. This is where the LLM judge is most uncertain (50% fail rate) and where expert
  signal has the highest information gain. Reviewing ~200 failures + ~100 passes gives us Cohen's kappa between expert and LLM judge — the key
  number we need to know whether the automated filter is trustworthy.
  2. Run volume ablation (50/100/200/300) on the cluster simultaneously — pure compute, no expert time needed. This tells us whether 50 filtered
   images is enough or we need all 300.
  3. Richard's lack of MA-specific expertise is manageable under Option B because he sees full metadata and the LLM judge's chain-of-thought
  reasoning — he can agree/disagree with specific morphological claims rather than identifying species from scratch.
  4. If kappa is high (>0.7): LLM judge is reliable, we can use it as the primary filter and Phase 3 crowdsourcing becomes a confirmation layer
  for publication. If kappa is low: we know exactly which failure modes the LLM misses and can fix the judge prompt or adjust the filtering
  threshold before scaling up.
