# 2026-03-18 — Pivot: Learning from Expert Evaluation

## Problem

Volume ablation (50–300 synthetic images/species) showed no statistically significant improvement on classifier performance. Bootstrap CIs overlap at 95% level. With only n=6 and n=10 test images for target species, we cannot reliably distinguish signal from noise.

Background removal test also showed the LLM judge is not the bottleneck — the real issue is **borderline coloration accuracy** in synthetic images, not background interference.

Current LLM-as-judge filtering helps (D5 > D4) but has a ceiling. Expert evaluation cannot scale (manual review of 1000+ images is infeasible). So the question becomes: **how do we learn the expert's signal and apply it at scale?**

## Core Idea

Collect expert evaluations on a small subset (~150 images: 50/species x 3 species), then use those labels to improve the filtering pipeline. Instead of using expert evaluation as a direct filter, use it as **training signal**.

### Expert Evaluation Plan

- 50 images per species (B. ashtoni, B. sandersoni, + 1 more TBD)
- 150 images total
- ~50 evaluations per reviewer per session (manageable workload)
- Binary pass/fail + optional notes on failure reason

## Potential Approaches to Learn from Expert Preferences

### 1. Preference-Tuned LLM Judge
- Collect expert pass/fail on 150 images
- Compare against current GPT-4o judge decisions
- Identify systematic disagreements (where expert fails but LLM passes, or vice versa)
- Fine-tune or prompt-engineer the judge to align with expert signal
- **Pro:** Stays within current pipeline, scalable
- **Con:** May not capture visual features experts use

### 2. Reward Model / Binary Classifier on Image Embeddings
- Extract embeddings (CLIP, DINOv2, or ResNet features) from the 150 expert-labeled images
- Train a lightweight binary classifier (pass/fail) on top
- Use as a learned filter replacing or augmenting LLM judge
- **Pro:** Directly learns visual quality signal, fast inference
- **Con:** 150 samples may be too few; needs careful validation

### 3. Preference Learning (Bradley-Terry / RLHF-style)
- Instead of binary pass/fail, collect **pairwise preferences** from experts (which of two images is better?)
- Train a reward model on preferences
- Rank all synthetic images by learned reward, keep top-k
- **Pro:** Richer signal per annotation, handles ambiguous cases
- **Con:** More complex annotation UI, 150 images → ~100 pairs per species

### 4. Active Learning Loop
- Start with expert labels on 150 images
- Train initial filter
- Use uncertainty sampling to select next batch of images for expert review
- Iterate until filter stabilizes
- **Pro:** Maximizes information per expert annotation
- **Con:** Requires multiple rounds of expert access

### 5. Contrastive Learning on Pass/Fail Groups
- Use expert-labeled pass/fail groups as positive/negative pairs
- Fine-tune image encoder to separate good from bad synthetic images
- **Pro:** Learns species-specific quality features
- **Con:** Small dataset, risk of overfitting

## Research Findings

### Key Insight

The filtering signal we need is **perceptual and fine-grained** (coloration accuracy, banding patterns), not semantic. The LLM judge reasons semantically ("does this look like B. ashtoni?"); experts reason morphologically ("is T4 tergite banding too orange?"). This gap is where **learned visual embeddings** have an advantage over prompted VLMs.

### Recommended Approach: DINOv2 Linear Probe (Primary) + LLM Judge Calibration (Complement)

**Why DINOv2 over CLIP:** DINOv2's self-supervised pre-training learns low-level texture and color statistics more faithfully than CLIP, which emphasizes semantics over fine-grained visual detail. Since the failure mode is coloration accuracy (a perceptual/texture signal), DINOv2 features will be more discriminative.

**Why linear probe, not fine-tuning:** With 150 samples, linear probing is more stable than fine-tuning (LP++, CVPR 2024). Fine-tuning a ViT-L backbone on 150 samples is high-variance.

**Also try:** BioCLIP (taxonomy-aware embeddings trained on TreeOfLife-10M) — may better separate morphological variants within Bombus.

### Stratified Sampling for Expert Annotation

Do NOT randomly sample 50 images per species. Use:
- 20 images: LLM judge passed, high morphological score (>= 4.0) — likely true positives
- 20 images: LLM judge near-boundary (score 3.5–4.0) — most informative
- 10 images: LLM judge failed — likely true negatives

This concentrates expert effort on the hardest cases and maximizes information about where the LLM judge is miscalibrated.

### Diagnostic Step: Expert vs LLM Judge Agreement (Cohen's Kappa)

Compute this BEFORE building anything:
- Kappa > 0.7: experts and LLM agree closely → prompt tuning won't help much
- Kappa 0.4–0.7: systematic disagreement → linear probe has something real to learn
- Kappa < 0.4: LLM is poorly calibrated → linear probe is essential

### Ensemble: Learned Probe + LLM Judge

Do not replace the LLM judge — combine:
```
final_score = alpha * learned_reward(x) + (1 - alpha) * llm_judge_score(x)
```
Sweep alpha in {0, 0.25, 0.5, 0.75, 1.0}. Calibrate on the 150 expert labels using logistic regression on both scores jointly (UrbanAlign-style post-hoc calibration, arXiv 2602.19442).

### Approach Comparison

| Approach | Data needed | Cost | Expected AUC | Recommend? |
|---|---|---|---|---|
| LLM judge prompt calibration | 150 labels | Low | Moderate | Yes, as complement |
| **DINOv2 linear probe** | **150 labels** | **Low (sklearn)** | **0.72–0.85** | **Primary** |
| Bradley-Terry on binary labels | 150 labels | Medium | ~same as probe | Variant of above |
| Contrastive fine-tuning | 150 labels | High (GPU, overfit risk) | Unknown | Only if probe fails |
| Active learning | 150+ labels | High (multi-round) | Potentially highest | Not feasible |

### Key References

- **UrbanAlign** (arXiv 2602.19442, 2025) — Post-hoc calibration of VLM using small human annotation sets
- **LP++** (CVPR 2024) — Linear probe on CLIP features achieves SOTA few-shot classification in 16–100 shot regime
- **PickScore** (Kirstain et al., NeurIPS 2023) — Fine-tune CLIP on human preference data with Bradley-Terry objective
- **BioCLIP** (arXiv 2311.18803) — Taxonomy-aware vision encoder, TreeOfLife-10M pre-training
