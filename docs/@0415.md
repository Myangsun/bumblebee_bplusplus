# Expert Validation Subset: Strategy Analysis

## 1. How the 150 Images Were Generated

### Algorithm (`scripts/build_expert_validation.py`)

**Input**: 1,500 LLM judge results (500 per species, from `RESULTS_kfold/llm_judge_eval/results.json`)

**Step 1 — Classify each image into an LLM quality tier:**

| Tier | Rule | Meaning |
|---|---|---|
| `strict_pass` | matches_target AND diag=species AND morph_mean >= 4.0 | LLM confidently IDs to species, high morphological scores |
| `borderline` | matches_target AND diag=species AND 3.0 <= morph_mean < 4.0 | Correct ID but moderate morph quality |
| `soft_fail` | matches_target AND diag < species | Correct genus but LLM can't confirm species-level ID |
| `hard_fail` | NOT matches_target | LLM identifies wrong species entirely |

**Step 2 — Allocate 50 slots per species, proportional to tier size (floor = 5 per non-empty tier).**

Each non-empty tier gets at least 5 images. Remaining slots distributed proportionally to tier pool size. Then `random.sample()` within each tier (seed=42). No angle or caste engineering — fully random within tiers.

**Step 3 — Copy selected images to `RESULTS_kfold/expert_validation/<species>/`**

## 3. Assessment: Is This Sampling Strategy Appropriate for the DINOv2 Probe?

### What the sampling does well

**The tier system already captures angle-quality correlations.** Frontal views that hide diagnostic features naturally fall into `soft_fail` (genus-level ID). Images with wrong coloration fall into `hard_fail`. The tier classification absorbs these quality dimensions — no need to manually engineer angle quotas or caste ratios on top. Random sampling within tiers inherits whatever angle/caste distribution the tier naturally has.

**Proportional-to-pool reflects the real distribution.** When the trained probe is applied to all 1,500 images, it encounters the true distribution (~64% strict_pass). Training on a sample that mirrors this distribution means the probe's priors match deployment. Artificially inflating negatives would bias the probe toward a distribution it won't see in practice.

### Potential concern: label imbalance for sandersoni

B. sandersoni has 37/50 strict_pass in the sample. If the expert labels most of these as pass, the probe has few negative examples for this species. But this reflects reality — sandersoni synthetics genuinely are 91% high quality.

**This is a modeling problem, not a sampling problem.** If label imbalance hurts probe performance, handle it at training time:
- Class-weighted loss (weight fail examples higher)
- Stratified train/val splits within LOO-CV
- Pool all species for the probe (150 images total, not per-species)

### Conclusion

**Keep the current `build_expert_validation.py` as-is.** The tier-proportional, random-within-tier sampling is sound for the DINOv2 probe use case. The angle distribution follows naturally from tier membership — no manual engineering needed. Label imbalance, if it's a problem, should be addressed in probe training, not by distorting the annotation sample.

---

## 4. The Harder Question: Will the Expert Filter Help?

The 150 annotations serve a **dual purpose**:

1. **Train the probe** (the stated goal)
2. **Diagnose WHY synthetic data doesn't help** — potentially more valuable

If the expert marks even the "best" strict_pass images as having subtle issues (wrong hair texture, unnatural body proportions), that points to **domain gap** — no filter will fix it.

If expert labels are clearly bimodal (some truly good, some clearly bad), then filtering has a shot.

The current sampling — covering all quality tiers proportionally — is well suited to answer both questions.
