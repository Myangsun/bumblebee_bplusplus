how to improve data representation

                                            
  - "Why not just use the LLM judge?" — It plateaued. 50% pass rate on B. ashtoni meant we couldn't tell if it was rejecting good images or passing bad ones. Expert labels gave us ground truth to diagnose this and train
  something better.                                                                                                                                                                                                               
  - "Why DINOv2 + linear probe instead of fine-tuning?" — Only 150 labeled images. A frozen foundation model + logistic regression avoids overfitting and trains in seconds. Same philosophy as the augmentation work: make the
  most of limited data.                                                                                                                                                                                                           
  - "How did you decide where to spend expert time?" — Stratified sampling across pass/fail/borderline — the borderline cases are where the LLM judge is uncertain and where expert signal has the highest marginal value.
  - "Statistical significance?" — Honest: per-species CIs overlap due to 6–10 test images. McNemar's on the full 2,350-image test set provides real power. Multi-seed runs separate training noise from method improvement.
