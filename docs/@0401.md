# 2026-04-01 — Expert Evaluation Plan

## How We Use Expert Evaluation

We collect structured annotations from one entomologist on 150 synthetic images (50/species, stratified by LLM judge score into high-pass/boundary/fail strata). The expert fills out the same rubric as the LLM judge — blind species ID, five morphological feature scores (1–5), diagnostic completeness, and failure mode checkboxes. This mirrors the LLM judge output exactly, enabling direct per-feature comparison.

**Step 1:** Per-feature disagreement analysis — compare expert vs. LLM scores on each dimension independently. Identify which features the LLM is miscalibrated on (e.g., consistently overscoring abdomen banding on B. ashtoni images that experts reject).

**Step 2:** Train a weighted feature-level filter. L2-regularized logistic regression on the 7 LLM rubric scores (5 morphological + blind ID match + diagnostic level) predicting expert pass/fail, learning the expert's implicit weighting over the LLM's dimensions. Validated via 5-fold CV on the 150 labels (report AUC-ROC).

**Step 3:** Apply the reweighted filter to all 1,100 synthetic images to assemble a new dataset (D6), which is used to retrain the ResNet-50 classifier (5 seeds, bootstrap CIs, McNemar's test against D5).

**Step 4:** The per-feature disagreement patterns feed back into a closed loop: (1) LLM judge prompt refinement with expert-derived species-specific criteria, and (2) generation prompt refinement targeting the specific morphological features that score lowest — diagnosing *why* synthesis fails and fixing it at the source, not just filtering after the fact.

**Alternative approach:** Train non-expert annotators (e.g., grad students) using a criteria card derived from the expert's calibration session, enabling cheaper scaling of annotation if more labeled data is needed.
